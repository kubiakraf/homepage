Repozytorium = wszystkie pliki projektu nad którymi pracuję

Commit - stan projektu i opis tego co się zmieniło od ostatniego zapisu

Dodajemy commita po większej porcji zmian, lepiej częściej niż rzadziej.

Command line

  • git config --global user.name "J0n Doe"
  • git config --global user.email "jondoe@example.com"
  • git status - polecenie do śledzenia zmian
  • git add. - dodanie zmian do commita
  • .git - tym katalogiem zarządza oprogramowanie do obsługi gita, nie ruszać

create a new repository on the command line:

  • echo "# 'repo'" >> README.md
  • git init - new repository
  • git add README.md
  • git commit -m "opis zmian" - comitowanie zmian
  • git branch -M main
  • git remote add origin https://github.com/"user"/"repo"
  • git push -u orign main

push an existing repository on the command line:

  • git remote add origin https://github.com/"user"/"repo"
  • git branch -M main
  • git push -u orign main

Podstawowy flow

  • git add.
  • git commit -m "opis zmian"
  • git push
