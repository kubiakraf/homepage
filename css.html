Definicja

Definiuje wygląd, prezentację na ekranach, czytnikach, wydrukach. Składnia CSS to reguły. Przeglądarki potrafią różnie interpretować CSS co prowadzi do różnic w wyświetlaniu.

selektor {
właścwości: wartość;
}

Deklaracja styli

  • inlinowe - ma jest nadżędny na innymi stylami, tylko jeśli nie możemy edytować pliku ze stylami bo są np. w zewnętrznej bibliotece.
    <znacznik style="name:wartość;">treść</znacznik>
  • w head pliku html
  • Deklarowanie pliku zewnętrznego ze stylami CSS w head strony:
    < link href="style.css" rel="stylesheet" type="text/css" >

Łączenie selektorów

po spacji = wszystkie a, które znajdują pośrednio lub bezpośrednio w section

section a {
właścwości: wartość;
}

Specyficzność w CSS

( _ , _ ) , _ , _ , _
!important style="" #name .name element

Dziedziczenie

selektor {name:wartość;}
!important - reguła umieszczana po deklaracji ale przed średnikiem czyni ją nadrzędną, ale jeśli ma być dziedziczony i nie jest zdefiniowany bezpośrenio do danego elementu nie jest brany pod uwagę. nadpisuje inne reguły dotyczące tej własności.
#id #(hash) - selektor id - nie używamy
.class .(kropka) - selektor klasy
element - selektor elementu

Jeśli specyficzność jest równa - pierwszeństwo ma selektor "bliżej" elementu, czyli niżej, na końcu pliku.

Rekomendowane jest dodawanie klas do wszystkich elementów i unikanie zagnieżdzeń. Nie dodawać styli za pomocą tagów (selektor elementu).

wartość: inherit;

Definiujemy tą wartością, że coś ma być dziedziczone. Jak np. color linków dziedziczony po <section> w którym link się znajduje:

section a {
color: inherit;
}

Nazewnictwo klas

  • pojedyńcze słowa z języka angielskiego
  • nazwy zgodne z ich funkcją na stronie
  • startuj z małej litery
  • nie zaczynać od cyfry
  • jeśli dwa słowa nie używamu spacji:
    • - camelCase - łączenie wyrazów zlepiając je i oddzielając wielkimi literami
    • - kebab-case - łączenie wyrazów myślnikiem
    • - snake_case - łączenie wyrazów za pomocą dolnej belki - nie używać bo nie zawsze
    • - PascalCase - zaczynamy od wielkiej litery i łączymy nimi kolejne wyrazy
  • starać się aby każdy element miał jedną klasę, unikać specyficzności
  • jeśli wybierzemy jakiś schemat nazewnictwa to trzymamy się go konsekwentnie

Podsumowanie klas

  • HTML: class="name" = CSS: .name
  • HTML: id="name" = CSS: #name
  • stylujemy po klasach (bez zagnieżdzeń)
  • nazwyamy klasy po angielsku zgodnie z przeznaczeniem
  • tworzymy klasy nawet jeśli mamy tylko jeden element danego typu

div

  • nie ma znaczenia semantycznego
  • gdy chcemy coś opakować i tylko kiedy nie istnieje odpowiedni semantyczny znacznik
  • przydaje się przy stylowaniu elementów
  • jeśli opakowujemy nim treść to prawdopodobnie robimy to źle

display: block

  • element zajmuje całą dostępną szerokość ekranu
  • jeśli nawet zajmuje mniej miejsca, rezerwuje tą przestrzeń
  • kolejne elementy będzie przerzucał do kolejnej linii

display: inline

  • element zajmuje tylko tyle miejsca ile wyświetlana w nim treść
  • kolejne elementy będą następować jeden po drugim w linii
  • wyświetlają się z delikatną przerwą jakby była między nimi spacja.
  • nie działa określanie width i height

display: inline-block

  • box który ma mniej treści zachowuje się jak liniowy
  • box któy ma więcej treści i wchodzi w kolejną linie staje się blokowym.
  • gdy przestał być całkowicie inline stał się blokowym
  • hybryda pomiędzy inline a block
  • elementy zachowują się trochę inlinowo ale można nadawać im szerokość i wysokość

Definiowanie zachowania bloków

width i height

  • sztywne rozmiary obiektu
  • jeśli się nie mieści na ekranie pojawia się scroll boczny
  • gdy węższy od ekranu pozostawia pustą przestrzeń

max-width i max-height

  • bloki zajmują całą dostępną zdeklarowaną przestrzeń
  • treść dostosowuje się do zmieniającej się szerokości ekranu
  • nie rozszerzy się bardziej niż maksymalna zadelkarowana

min-width i min-height

  • gdy ekran się zmniejsza poniżej zadeklarowanej wartości minimalnej pojawia się boczny scroll
  • element nie może być mniejszy niż zadeklarowana
  • zachowują się trochę jak elementy blokowe

overflow

  • definiuje jak ma się zachować element gdy treść zajmuje więcej miejsca niż blok
  • : hidden; - wizulanie ucina treść która się nie mieści
  • : auto; - tworzy scrolla aby można przewinąć treść - tylko tam gdzie jest potrzebny
  • : clip; - na tym etapie działa jak hidden ale przydaje się przy bardziej zaawansowanym stylowaniu
  • : scroll; - wszystkie elementy, niezależnie czy potrzebują mają pasek do przewijania
  • : visible; - treść wystaje po za blok, tak jakby overflow nie był definiowany
    w tym przypadku treść nachodzi na treść elementu poniżej, nie respektuje jego przestrzeni

jednostki

zapisywać bez spacji

  • 0 - bez jednostek, zero to jest zero nieważne czego
  • px - piksele - jeden punkt na ekranie
    przeliczony przez przeglądarkę a nie ten ze specyfikacji technicznej monitora
  • % - procenty - wartość odnosząca się do wielkości rodzica.
    element wyżej musi mieć zdefiniowaną wysokość aby móc wyrazić ją w procentach
  • vw - viewport width - szerokość okna w którym wyświetla się dokument w procentach
    nie odnosi się do rodzica, zawsze do okna ekranu (posiada margines)
  • vh - viewport height - bierze całą wysokość okna (nie całości dokumantu)
    gdy treść sekcji ma być na cały ekran jest to robione za pomocą height: 100vh;

Box model

margin
border
padding
element

Wszystkie elementy na stronie są otoczone "niewidzialnym prostokątem"
Jest widoczny gdy podświetla się go w Inspect

  • margin

    • Określa o ile element ma się odsunąć od innych elementów, które go otaczają.
    • Zapisuyjemy tak samo jak padding.
    • marginesy nakładają się na siebie.
    • elementy są od siebie odsunięte o wyższą wartość, mniejsza jest jako minimalna.
    • margin: __px auto; pierwsza wartość do top-bottom a druga auto = wyśrodkowanie elementu

  • border

    • Zapis pełny:

      • border-width: 3px;
      • border-style: solid;
      • border-color: #fe201a;

    • Zapis skrótowy:

      • border: 3px solid #fe201a;

    • wybór fragmentu ramki:

      • border-top: 3px solid #fe201a;
      • border-right
      • border-bottom
      • border-left

    • border-radius:

      • Odpowiada za zaokrąglenie.
      • px lub %
      • Nie tzeba definiować border aby mieć zaokrąglony box.

  • padding

    • definiuje o ile nasza treść ma być oddalona od ramki, krawędzi tego elementu

    • Zapis pełny:

      • padding-top: 4px;
      • padding-right: 4px;
      • padding-bottom: 5px;
      • padding-left: 6px;

    • Zapis skrótowy (shorthand):

      • padding: top right bottom left;
      • padding: 4px 4px 5px 6px;

    • Wartości lustrzane:

      • padding: top right-left bottom;
      • padding: 4px 5px 6px;

    • Drugi zapis skrótowy:

      • padding: top-bottom right-left;
      • padding: 4px 5px 6px;

    • Wszystkie wartości są takie same:

      • padding: 16px;

  • box-sizing: border-box;

    • sprawia, że podany wymiar width i height jest traktowany jako całkowity aż do border
    • element + padding + border = zdeklarowany width i height
    • standardowa wartość to content-box;

  • position: static;

    • domyślna wartość - przeglądarka sama decyduje o układzie elementu (na podstawie jego parametrów)

  • position: relative;

    • pozwala stworzyć pewnego rodzaju przestrzeń, względem której możemy umieszczać inne elementy
    • samo dodanie nic nie zmienia wizulanie ale pozwala na wykorzystanie przestrzeni

  • position: absolute;

    • absolutne pozycjonowanie względem elementu z relative
    • top: left: right: bottom:
    • top: 0px - element będzie przyklejony do krawędzi paddinga
    • można dodać padding do elementu

  • position: fixed;

    • odpawiada za przyklejenie elementu do vievportu - otwartego okna
    • i pływa razem ze stroną
    • jest ciągle przyklejony w pozycji: top, left, right, bottom
    • gdy ma z-index: -1; podczas przesuwania chowa się za elementami
    • używany w urządzeniach mobilnych do przyklejania przycisku np. menu

  • position: sticky;

    • bardzo przydatna własność szczególnie przy dłuższych tekstach i chcemy przykleić nagłówek wewnątrz kontenera
    • nie zadziała bez własności zdefiniowanej własności top
    • jeśli rodzić ma własność overflow to sticky nie zadziała
    • jest przyklejony w pozycji: top, w miejscu w którym był jako np. nagłówek
    • pływa razem ze stroną do momentu aż dojdzie do końca elementu gdzie się zatrzymuje
    • działa tylko wewnątrz rodzica

  • float

    • element w divie jest opływany przez tekst jak np w wordzie
      • width: 100px;
      • height: 100px
      • background: red;
      • margin-right: 20px;
      • float: left;
    • margines diva, który jest automatyczny na całą szerokość jest zastępowany przez ten który definiujemy
    • nie używamy float do budowania layoutów
    • tylko do opływania img

  • clear

    • czyści floaty
    • clear: both; - czyści lewy i prawy

  • Z-index

    • odpowiada za ułożenie warstw
    • im wyższy index to element będzie wyżej
    • tło może mieć wartość -1
    • domyślnie ten element, który jest niżej w kodzie jest wgrywany ostatni na stronę i może nadpisać wcześniejsze
    • Microsof Edge ma wbudowane narzędzie do podglądu Z-index
      • uruchomić dev tools
      • Ctrl + Shift + P
      • 3D View

Ujednolicanie styli - przeglądarki posiadają własne style (user agent stylecheet)

  • Reset - stara technika
    • * { - selektor uniwersalny, wybiera wszystkie elementy na stronie
    • margin: 0;
    • padding: 0;
    • outline: none; - niewidać, na jaką wartość przeskoczyło się tabulatorem = problem dla ludzi słabowidzących
    • }
  • Normalizacja - normalize.css
    • Elementy będą wyświetlać się na różnych przeglądarkach bardziej spójnie, zgodnie z najnowszymi standardami.
    • skopiować treść i dodać do projektu jako normalize.css

    • html {
    • box-sizing: border-box;
    • }

    • *, *:before, *:after {
    • box-sizing: inherit;
    • }

BEM - block element modifier

  • Block - nadrzędny element
  • Element - część bloku (nie musi być bezpośrednim dzieckiem)
  • Modifier - modyfikator - jeśli element występuje w kilku wariantach tworzymy dodatkową klasę.
  • block__element--modifier
  • Nadajemy klasami sens semantyczny odwzorowując strukturę strony
  • zaczynamy od stylu blockowego (nagłówek)
  • następnie style elemtów (rozdziały)
  • a pod każdym elementem modyfikatory (pod rozdziały)

  • Szukamy elementów na stronie i dzielimy na bloki.
  • Myślimy o elementach składowych / robimy analizę-selekcję.
  • Tworzymy dwie klasy jeśli element ma modyfiaktor (w modyfiaktorze zmieniamy tylko unikalne dla niego wartości, nie przepisujemy rzeczy z klasy bazowej).

Typografia

  • font-size - rozmiar tekstu
  • font-family - krój fontu, bezszeryfowe do nagłówków, a szeryfowe do tekstu.
  • font-weight - grubość
  • line-height - wysokość odległości między wierszami.
  • letter-spacing - odstępy między literami w wyrazie

Typografia - jednostki

  • 1em = font-size dla elementu (dziedziczony / ustawiony) *1
  • 1rem = font-size dla root elementu *1
  • (domyślnie przeglądarka nadaje 16px = root, pobierany z htmla)
  • https://nekocalc.com/px-to-rem-converter - kalkulator px na rem
  • zostawić lub ustawić font-size na 16px
