HTML5 - Hyper Text Markup Language
(Hipertekstowy język znaczników)

Definicja

Nie jest językiem programowania, jest językiem znaczników. HTML5 stał się tak zwanym "living standard" - wersja 5 będzie rozwijana, nie powstanie nowa. Odpowiada za treść nie za wygląd. Opakowuje i opisuję treść strony. Treść musi być semantyczna, stylowanie obsługuje CSS. Mówi przeglądarce o strukturze dokumentu i o tym co ma wczytać - podział na sekcje, artykuły, nagłówki - to jest struktura. Posiada swoją zdefiniowaną składnię - znaczniki, które zawierają w sobie treść.

znacznik/tag/elemetn

<znacznik atrybut="wartość">treść</znacznik>

Semantyczny - mający znaczenie, znaczniki nadają znaczenie treści, którą opakowują.

Podstawowa struktóra dokumentu HTML

  • <!DOCTYPE html> - typ dokumnetu
  • <html lang="en"> - korzeń strony i atrybut języka strony, ważne dla czytników ekranowych
  • <head> - dodatkowe informacje dla przeglądarki potrzebne do załadowania strony oraz silników wyszukujących
  • <meta charset="UTF-8"> - kodowanie znaków dla całego świata
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> - info o responsywności
  • <title>Document </title> - tytuł dokumentu na zakładce w przeglądarce
  • </head>
  • <body>
  • <main>
  • <header>
  • <h1>Nagłówek strony</h1>
  • </header>
  • <section>
  • <h2></h2>
  • <p>treść paragrafu 1</p>
  • </section>
  • <section>
  • <h2></h2>
  • <article>
    • <h3></h3>
  • <p>treść paragrafu artykułu</p>
  • </article>
  • <article>
    • <h3></h3>
  • <p>treść paragrafu artykułu</p>
  • </article>
  • </section>
  • </main>
  • <footer>
  • <p></p>
  • </footer>
  • </body>
  • </html>

main

Unikatowa treść strony, nie występująca na innych podstronach. Główna część dokumentu.

header

Nałówek - dokumentu (main) / sekcji (section) / artykułu (article) Kontener, pudełko na nagłówki.

Jeśli zawiera nav to musi być poza main gdyż powatarza się na wszystkich stronach i nie jest unikatową treścią. Kolejny header może być nagłówkiem main.

nav

Znacznik opakowujący nawigację serwisu. Umieszcza się go w <header>. Opakowuje główną nawigację między stronami.

article | section

Sekcja to jakby dział sklepu, półka. Może mieć w sobie kilka artykułów. Zawiera w sobie ten sam rodzaj treści.
Artykuł to tak jakby artykuł w sklepie na półce. Samodzielna treść. Jedne artykuł np. na blogu, może mieć w sobie sekcje i zaczynają się one od nagłówka.

article

  • do oznaczania niezależnych, samodzielnych elementów zawartości, które mają sens w oderwaniu od reszty strony
  • pojedynczy artykuł, wpis na blogu, komentarz, wiadomość itp.
  • powinien zawierać treść, która stanowi samodzielny byt

section

  • używany do logicznego podzielenia zawartości strony na sekcje
  • może zawierać różne rodzaje zawrtości: tekst, obrazy, listy itp
  • cel - pomóc w organizacji strony i podzielenie na spójne części
  • gdy mam treść, którą można logicznie pogrupować
  • gdy nie są na tyle niezależne, aby stanowić osobne artykuły

footer

Stopka - dokumentu / sekcji / artykułu

Jeśli mamy wiele pod stron to <footer> musi znaleść się za <main>, bo powiela się na wszystkich stronach

h1-h6

Nagłówek tekstowy - dokumentu / sekcji / artykułu W dokumencie od największego do najmniejszego. Opakowany przez header jeśli są dodatkowe info dotyczące nagłówka np. data publikacji, autor.

p - paragraf

Blok tekstu.

<br /> - break line

Złamanie tekstu, nie trzeba już go zamykać za pomocą "/"

Elementy do wyróżnienia tekstu

  • <b> - bold - brak znaczenia semantycznego, wyróżnia wizulanie, nie używać, teraz używa się CSS
  • <i> - italic - brak znaczenia semantycznego, wyróżnia wizulanie, nie używać, teraz używa się CSS
  • <u> - under line - działa ale teraz CSS, nie używać
  • <strong> - bold i położenie akcentu, podkreśla wysoką istotność i poważność frazy
  • <em> - italic i podkreśla, że tekst może mieć inne znaczenie (np. ironia)

span

  • brak znaczenia semantycznego
  • element liniowy
  • głównie do opakowywania tekstu i stylowania

Listy

  • <ul> - unordered list - lista nieuporządkowana
  • <li> - list item
  • <ol> - ordered list - lista uporządkowana
  • list-style(-type): disc, none, circle, squere
  • <dl> - description list
  • <dt> - description term
  • <dd> - description details

Tabele

  • <table> - znacznik otwierający tabelę
  • <tr> - table row - wiersz tabeli
  • <td> - table data - komórka tabeli
  • <thead> - nagłówek tabeli
  • <th> - komórka w nagłówku tabeli
  • <tbody> - ciało tabeli
  • <caption> - opis tabeli

a - anchor (kotwica)

  • <a href="https://..." target="_blank" >"opis"</a>
  • <a href="#name" >"chapter 1"</a> przenosi do elementu <zancznik id="name">
  • Używamy do linków oraz do łącz wewnątrz dokumentu.
  • target="_blank" - sprawia, że link otwiera się w nowym oknie

atrybut id - identyfikuje element

  • Przydaje się do nawigacji w obrębie dokumentu, nie może się powtarzać bo jest unikalny.
  • <element id="moj-identyfiaktor"> - id identyfikuje dowolny element na stronie
  • <a href="#moj-identyfikator"> - link do zdefiniowanego wyżej elementu na stronie

img - image

<img src="*.png" alt="alternativ text"/>

Jest to znacznik samozamykający się.

/ - przed destynacją obrazka jest odwołaniem do korzenia strony i zaczynaniem szukania od poziomu 0.
Brak / z przodu ścieżki czyni adres zasobu relatywnym.
Jeśli dodam go w stronie robionej na dysku to będzie odwoływać się do początku dysku C.

Relacje między elementami

  • Głómny element html to korzeń,
  • dalej są zagnieżdżenia, które mogą mieć w sobie kolejne zagnieżdżenia.
  • Element zagnieżdżony jest dzieckiem elementu w którym jest zagnieżdżony czyli rodzica (powszechna metoda opisywania relacji między elementami).
  • Pojedyńczy element nazywamy węzłem lub nodem.

DOM - Document Object Model

  • Plik html ani źródło strony(view source) to nie jest DOM
  • to co widzimy w devtoolsach to jest DOM - to taki złożony w przeglądarce HTML
  • plik HTML to instrukcja - to co w devtoolsach to działająca strona (skręcony mebel)
  • DOM to taki pomost między HTML a JS
  • JavaScript może modyfikować DOM
  • funkcja querySelector zwraca node (węzeł) z HTML-a, selektor jest zapisywany zgodnie z CSS
  • korzystając z BEM - tworzymy dodatkową klasę z mofygikatorem --js (block__element--js)
    (ułatwia identyfikację / wyklucza potencjalne problemy)
  • document.querySelektor('.selector') ← Tak samo jak w CSS!
  • main.js:
    • const header = document.querySelector('h1')
    • console.log(header) - odnosi się do elementu
    • => w Inspect w Console pojawia się węzeł dokumentu:
    • ►h1.page-header__heading.page-header__heading--js :
    • ▼ lista pól i funkcji do wykorzystania na tym elemencie
    • dodanie "defer" przed "src" do polecenia wczytania JavaScriptu powoduje, że Consola zwraca nam to co w sourceview
    • ><script defer src="main.js"> </script> :
    • < h1 class="h1.page-header__heading page-header__heading--js">Treść nagłówka z kodu html</h1>
    • klasy zostały nadane zgodnie z BEM w kodzie
    • const header = document.querySelector('.page-header__heading--js')
    • wybór za pomocą selektora
  • nazwaZmiennejQuerySelector.innerHTML = 'treść'
    • własność innerHTML odpowiada za treść elementu
    • console.log(header.innerHTML) = wyświetli w Console aktualną treść headera - podaje wartość a nie element
    • header.innerHTML = 'string' - podmienia wartość, którą przeglądarka wyświetli - bo js nadpisuje html
    • np:
    • header.innerHTML = 'Nazywam się ${firstName} i mam ${age} lat.' - będzie się wyświetlać w h1 na stronie
    • console.log(header.style) = w Console: ►CSSStyleCeclaration - wszystkie własności CSSowe które są możliwe i można je modyfikować:
    • header.style.color = 'red'; - wpisanie w main.js nadpisze inlinowo w html, styl ccsowy w DOM
    • w kodzie nie widać bo widoczne w DOM i oczywiście generuje problemy

Formularze

Umożliwiają kontakt ze światem i wysyłanie danych.

type="text"

  • <form> - otwarcie formularz, wyznacza jego granice
    • <p>
    • <label for="firstName">Imię</label> - etykieta do pola formularza z opisem
      • powiązanie label z idPola - for==id
      • po kliknięciu w nazwę pola można pisać w inpucie
    • <input type="text" id="firstName" name="firstName" value="Rafał" placeholder="Wpisz imię"/>
      • <input /> - znacznik samozamykający, najczęściej opakowany w znacznik p, aby było w miarę prawidłowo semantycznie
      • type="text" - podstawowy typ inputa, jednowierszowy text
      • id="idPola" - używany do identyfikowacji pola, zwykle takie samo jako nazwa pola, ale to nie to samo
      • name="nazwaPola" - atrybut inputa, łączy ze sobą w radio ze sobą pozycje z fomrularza, np. aby nie można zaznaczyć kilku w radio
      • value="wartość" - (domyślna wartość w inpucie wpusuje w okno), wysyła ją na serwer
      • placeholder="wyszarzona podpowiedź" - wyświetla się gdy inut jest pusty, nie zastępuje labela
      • checked - w polach wyboru ta opcja będzie domyślnie zaznaczona
    • </p>
  • </form>
